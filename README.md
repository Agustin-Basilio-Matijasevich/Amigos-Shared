# Amigos-Shared
Sirve como medio para pasarnos codigo entre amigos y experimentar ser colaboradores de un repositorio con github, el objetivo es compartirnos codigo pero si alguno quiere ponerse a programar en su carpeta puede hacerlo mientras sea en su carpeta no hay drama

# Note
Por Favor:

&nbsp;
EDITAR el archivo .gitignore para evitar subir archivos innecesarios segun los lenguajes que utilicen y añadir al readme un registro de la configuracion del ignore, si van a cambiar el gitignore o el readme antes actualicen el que esta en su rama haciendo merge de la main a su rama asi se aseguran que estan modificando sobre el archivo actualizado.

&nbsp;
Si realizan modificaciones en la carpeta de otro por acuerdo entre pares o modifican el gitignore o el readme hagan el commit en su arbol y un pull request inmediatamente
aclarando de esos cambios y avisen al resto asi nos descargamos los cambios en nuestros arboles (obvimente el resto hagan el merge a sus arboles para descargarse los cambios), 
no importa si hay otros cambios hechos dentro de su carpeta en el mismo pull request mientras sean en su carpeta pasan de una; Esto es para evitar tener problemas en futuros merge ya que 
si alguno añade las excluciones para un lenguaje nuevo y otro tambien se pone a programar en ese lenguaje haciendo las mismas modificaciones cuando el primero haga un pull request no va a haber problema se va a cargar al main de una, 
pero cuando el segundo haga el pull request de su arbol al main va a haber un conflicto probablemente, y si no puede pasar que se ensucie el readme o el gitignore con codigo y texto de mas.

&nbsp;
Pongan su codigo dentro de sus carpetas y utilicen sus correspondientes ramas.

&nbsp;
Si quieren crear una rama extra temporal haganlo pero al terminar de usarla hagan merge a su rama y borrenla del server.

&nbsp;
NO se metan en la rama de otros ni hagan pull request para modificar la rama de otro las pull request son para el main unicamente.

&nbsp;
Cuando quieran subir su codigo a la rama main hagan un pull request y yo lo reviso antes.

&nbsp;
Mientras hagan modificaciones solo dentro de su carpeta no tengo probelma en que los pull request sean grandes.
